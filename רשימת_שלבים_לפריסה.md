# רשימת שלבים לפריסה לרשת (10 שלבים)

| # | מה עושים |
|---|----------|
| 1 | חשבון GitHub (אם אין) |
| 2 | יצירת Repository חדש – שם כמו `webtarbutu` |
| 3 | העלאת הקוד מהמחשב ל-GitHub (ב-PowerShell או GitHub Desktop) |
| 4 | חשבון Render + התחברות עם GitHub |
| 5 | יצירת Web Service – חיבור למאגר, Build: `npm install`, Start: `npm start` |
| 6 | הוספת משתני סביבה (העתקה מ-.env ל-Render) |
| 7 | Create Web Service → לחכות ל-Live → להעתיק את הכתובת |
| 8 | ב-Twilio – להגדיר Webhook לכתובת מ-Render |
| 9 | כניסה ללוח ניהול: `https://הכתובת/admin` |
| 10 | לתת למפתח האתר את קוד הווידג'ט עם הכתובת |

---

**אחרי שלב 7** – שמור את הכתובת (למשל `https://webtarbutu-xxxx.onrender.com`).

**שלב 10 – קוד למפתח האתר:**
```html
<script>
  window.WEBTARBUTU_API = 'https://הכתובת-מ-Render';
  window.WEBTARBUTU_AGENCY = 'תרבותו';
  window.WEBTARBUTU_WHATSAPP_NUMBER = '972XXXXXXXXX';
</script>
<script src="https://הכתובת-מ-Render/widget.js" async></script>
```
